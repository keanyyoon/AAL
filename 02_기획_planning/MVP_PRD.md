# MVP PRD — Q4 2026 파일럿 (Product Requirements Document)

- 작성일: 2026-07-01 · 작성: AAL 기획팀 · 상태: Draft(컨펌 대기)
- 범위: 첨부 'MVP 화면 정의'의 **P0 화면 4종**을 단일 진실 소스(SSOT)로 정의. 스코프 밖은 §7.

## 0. 목표와 비목표
- **목표(Goal):** 8~12주 파일럿에서 4대 데이터를 신뢰성 있게 확보하는 최소 제품.
- **비목표(Non-Goal):** 초기 진단(레벨테스트), 튜터 선택 UX, 또래 소셜/보상 대규모 시스템, KT(Knowledge Tracing) 반영 리포트, 결제 — 모두 Phase 1+ (§7).

## 1. P0 화면 4종 개요
| ID | 화면 | 사용자 | 측정 데이터 |
|---|---|---|---|
| SM | 세션 메인 (지문+문항+AI 대화) | 학생 | ① 완료율 ② 페르소나 ④ 학습패턴 |
| SE | 세션 종료 + 학생 리포트 | 학생 | ① 완료율 ② 페르소나 |

## 2. SM — 세션 메인 화면
**목표:** 지문 읽기 → 문제 → 학생 응답을 한 화면에서 통합 진행. AI는 학습을 주도하지 않고 **막힌 지점에만 소크라틱 발문**으로 메타적 개입.
**구성:** ①지문+문항 패널(레벨 일부 운영) ②AI Agent 대화 패널(발문→응답→비계, 답은 안 줌) ③페르소나 로깅 ④문항 유형·난이도 응답 패턴 기록.

**유저 스토리**
- 학생으로서, 지문을 읽고 문제를 풀다 막히면, AI의 질문을 통해 스스로 실마리를 찾고 싶다.
- 학생으로서, 답을 바로 얻지 않아도 좌절하지 않게 단계적 힌트(비계)를 받고 싶다.

**수용 기준(Acceptance Criteria)**
- [ ] AI는 어떤 경우에도 정답을 직접 제공하지 않는다(가드레일). 대신 발문·비계 사다리(①조건 표시 →②결과 표시 →③연결 힌트)로 유도.
- [ ] 학생 정체(무응답/오답 반복) 감지 시 비계 단계가 자동 상향된다.
- [ ] 문항별 정오답·유형·난이도·소요시간·비계 사용 횟수가 로깅된다(개발팀 스키마 연동).
- [ ] 현재 세션의 페르소나 ID가 모든 상호작용에 태깅되어 로깅된다.
- [ ] 태블릿·PC에서 지문/대화 패널이 반응형으로 배치, 모바일은 세로 스택.

**성공지표:** 세션 내 문항 시도율, 비계 사용 후 정답 도달률, 관찰 사고행동(근거 대기·텍스트 복귀) 발생 횟수.

## 3. SE — 세션 종료 + 학생 리포트
**목표:** 완료 버튼 클릭 순간 완료 데이터 자동 기록 + 학생 눈높이 톤으로 성장 즉시 피드백.
**구성(3블록):** ✓오늘 배운 것 / ↑지난 레슨 대비 좋아진 점(구체 근거: "근거 지난번 1개→오늘 3개") / →조금 더 보강할 부분(다음 세션 예고 연결).

**수용 기준**
- [ ] 완료 버튼 클릭 = 세션 완료 이벤트 + 전체 로그 확정(완료율 KPI의 원천).
- [ ] "좋아진 점"은 실제 로깅된 사고행동 변화에 근거해 생성(임의 칭찬 금지).
- [ ] Endorse 톤이 페르소나별로 구분 적용·로깅된다(파일럿 A/B).
- [ ] 접속 시: 아이스브레이킹(학습 외 관심사) + 지난 레슨 복기 + 오늘 프리뷰.

**성공지표:** 완료 버튼 도달률(=완료율), 종료 리포트 열람 후 다음 세션 재접속률.

## 4. PD — 학부모 데일리 세션 로그
**목표:** 매 세션 후 자동 업데이트되는 데일리 로그(알림은 미발송).
**구성:** 학습 지문 난이도 / 강한·약한 문항 유형 / 선호 난이도·유형 패턴 / 세션 시간·완료 여부.
**수용 기준**
- [ ] 매 세션 종료 시 자동 갱신. [ ] 언제든 열람 가능. [ ] 성과 알림만 앱/카카오 발송(데일리는 무알림).

## 5. PT — 학부모 4주 성장 추이
**목표:** 4주 단위 추이 차트 + AI 코멘트로 "성장의 서사" 전달 → 구독 갱신 동기(파일럿에선 갱신 대신 만족·재참여로 측정).
**구성:** 고난이도 지문 이해 / 강점 문항 적중률 / 약점 문항 개선도 (1–4주/5–8주/9–12주) + AI 코멘트. 내부 난이도 ↔ 외부 기준(CEFR·수능/내신 등급) 매핑.
**수용 기준**
- [ ] 4주 경계마다 추이 갱신(8~12주 = 2~3회 비교 가능). [ ] AI 코멘트는 데이터 기반 생성. [ ] 내부 레벨-외부 준거 매핑 표기.

## 6. 성과 알림 규칙 (공통)
성과 포인트에서만 발송(피로 최소화): 새 난이도 지문 첫 이해 / 지식·논리·의미 3축 중 성장 시 / 주간 스트릭 달성.

## 7. 스코프 밖 (Phase 1+)
초기 진단(레벨테스트), 튜터 선택 UX, 또래 소셜·대규모 보상, KT 반영 리포트, 결제. → 파일럿은 페르소나 1~2개(또는 A/B용 다수) 고정, 난이도 데이터로 레벨 관찰 대체.

## 8. 성공 지표 ↔ 4대 데이터 매핑
① 완료율(SM·SE) · ② 페르소나 반응(SM·SE, 4주 로테이션 A/B) · ③ 리포트 반응(PD·PT 열람률·알림 오픈율·섹션 체류) · ④ 학습 패턴(SM 로깅 → PT 추이).
