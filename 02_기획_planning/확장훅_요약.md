# 확장 훅 요약 (Expansion Hooks) — 지금 정해야 나중에 후회 안 할 결정

- 작성일: 2026-07-01 · 작성: AAL 기획팀 · 원칙: "속도를 내는 가장 확실한 방법은 되돌아올 일을 만들지 않는 것"
- 성격: 일정·비용 부담은 낮으나 나중에 바꾸면 비싼 **설계 단계 결정**만 모음. 개발팀·UX팀에 반영 요청.

## 1. 데이터/모델 구조
- **언어독립 사고요소 층 분리:** 지식그래프에서 논리(Logic)·지식②(재구성)를 언어 비종속 층으로 분리 저장 → 국어·수학 재사용. 의미(Meaning)·지식①(배경)은 언어 종속.
- **사고행동 로깅 스키마 과목 비종속:** 근거 대기·텍스트 복귀·사실/의견 구분 등을 과목 태그와 분리해 기록.
- **학생 성장 기록의 영속성:** 콘텐츠·과목이 바뀌어도 성장 로그는 학생 단위로 누적(락인 자산).

## 2. 다국어/글로벌 골격
- **i18n 전제:** UI 문자열·콘텐츠·튜터 발화의 다국어 분리(하드코딩 금지). 대만(번체)·베트남어 대비.
- **국가별 운영·결제·가격 체계** 자리만 확보(파일럿은 미사용).
- **외부 준거 매핑 테이블:** 내부 레벨 ↔ (CEFR / 수능·내신 / TOEFL Junior / 현지 시험) 매핑을 데이터로 관리(코드 하드코딩 금지) → 지역별 준거 교체 용이.

## 3. 리포트/평가
- **두 갈래 리포트(성취+사고) 구조 고정:** 국어·수학 확장 시 사고 축 재사용.
- **구인 타당도 대비:** 원거리 전이(주제 무관) 측정 훅을 데이터 모델에 예약.

## 4. 페르소나/세계관
- **페르소나를 세계관 속 캐릭터로 정의:** 과목·언어 확장 시에도 동일 세계관·캐릭터 유지 가능하게 추상화(브랜딩 인계).

## 5. 연령 확장
- 난이도·레벨을 연령 하드코딩이 아닌 **마스터리 기반**으로 → 위/아래 연령 개방 용이.

## 6. "지금은 안 하지만 막지 않을 것" 체크리스트 (개발/UX 수용기준화)
- [ ] 모든 사용자 노출 문자열 i18n 분리
- [ ] 사고요소 언어독립 층 분리
- [ ] 로깅 스키마 과목/언어 비종속 필드 포함
- [ ] 내부레벨-외부준거 매핑 테이블화
- [ ] 성장 로그 학생 단위 영속 저장
- [ ] 페르소나-세계관 추상화

> 검증팀: 위 6개 항목이 UX/개발 산출물에 반영됐는지 정합성 리뷰에서 점검.
