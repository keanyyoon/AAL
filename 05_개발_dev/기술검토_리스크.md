# 기술 검토 & 리스크 (Feasibility & Risks)

- 작성일: 2026-07-01 · 작성: AAL 개발팀

## 1. Q4 파일럿 타당성 (개발 범위)
- P0 4화면 + 온보딩/로그인 + 성장맵/리워드는 반응형 웹 SPA로 **파일럿 규모(100명, 8~12주)에 구현 가능**한 범위.
- 최대 난이도: ①소크라틱 대화 품질/가드레일 ②로깅 무결성 ③리포트 AI 코멘트 품질.

## 2. 리스크 & 완화
| 리스크 | 영향 | 완화 |
|---|---|---|
| 정답 유출(가드레일 실패) | 정체성 붕괴·교육효과 훼손 | 이중 가드레일(프롬프트+후처리) + 레드팀 QA 상시 |
| 대화 지연 | 이탈·완료율↓ | 스트리밍, 경량 상태추정, 캐싱 |
| 로그 누락/불일치 | 파일럿 데이터 무효 | 완료버튼=트랜잭션, 스키마 검증, QA 데이터 무결성 테스트 |
| 사고행동 태깅 부정확 | ④·리포트 신뢰도↓ | 보수적 규칙+AI, 파일럿 중 정밀도 튜닝 |
| LLM 비용/변동 | 운영비·안정성 | 파일럿 규모 제한, 모델 추상화(교체 용이) |
| 아동 개인정보 | 규제·신뢰 | 최소수집·가명화·동의, 검증팀 점검 |
| 측정 타당도(영어실력 그림자) | 문해력 주장 신뢰 | 원거리 전이 데이터 훅, 외부 검증 |
