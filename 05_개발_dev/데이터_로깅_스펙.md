# 데이터 로깅 스펙 (Data Logging Spec) — 파일럿 4대 데이터 보장

- 작성일: 2026-07-01 · 작성: AAL 개발팀 · 목적: 4대 데이터가 반드시 산출되도록 이벤트 스키마 정의

## 1. 핵심 엔티티 (관계형)
- **user**(id, role[student/parent], persona_current, locale, created_at, 청담_ref)
- **session**(id, user_id, persona_id, started_at, completed_at, completed[bool], duration_sec)
- **item_result**(id, session_id, item_id, type, difficulty, is_correct, time_sec, scaffold_used[int])
- **thinking_event**(id, session_id, kind, ts) — kind ∈ {근거대기, 텍스트복귀, 사실의견구분, 반박수용, 추론시도}
- **persona_interaction**(id, session_id, persona_id, turn_no, endorse_tone, student_reaction?)
- **report_view**(id, parent_user_id, report_type[daily/trend], section, dwell_sec, ts)
- **notification**(id, user_id, trigger[new_difficulty/axis_growth/streak], opened[bool], ts)

## 2. 4대 데이터 ↔ 로그 매핑
| 데이터 | 산출 쿼리 원천 | 무결성 규칙 |
|---|---|---|
| ① 세션 완료율 | session.completed / 전체, 완료까지 duration | **완료 버튼 = 단일 트랜잭션으로 session.completed + 모든 item/thinking 이벤트 확정** |
| ② 페르소나 반응 | persona_id별 완료율·성취·duration + 종료설문 | 모든 대화 턴에 persona_id 태깅 필수 |
| ③ 리포트 반응 | report_view(열람·섹션 체류), notification(오픈율) | 섹션 단위 dwell 로깅 |
| ④ 학습 패턴 | item_result(난이도·유형·정오답·시간), 4주 집계 | 문항 메타(type/difficulty) 정규화 |

## 3. 이벤트 발생 지점(클라이언트→서버)
- 로그인 시각 / 세션 시작 / 문항 제출(정오답·시간) / 비계 사용 / 사고행동 감지 / 페르소나 턴 / 완료 버튼 / 리포트 열람·섹션 체류 / 알림 오픈.

## 4. 사고행동(thinking_event) 감지 방식
- 규칙 기반 + AI 판정 병행: 예) "텍스트로 돌아가 인용" → 텍스트복귀, "근거 문장 제시" → 근거대기. 초기엔 보수적으로 태깅하고 파일럿에서 정밀도 검증.

## 5. 프라이버시·안전
- 아동 개인정보 최소 수집(식별정보 분리 저장·가명화). 학습 로그는 분석 목적. 보호자 동의·열람권.

## 6. 확장 훅
- item_result·thinking_event에 **subject / language 필드**(과목·언어 비종속) 예약 → 국어·수학 재사용.
- 사고요소는 언어독립 층 참조 키 보유.
- 내부레벨-외부준거 매핑은 별도 테이블(mapping) 참조.
- 성장 로그는 user 단위 영속(콘텐츠 교체 무관).

> 검증팀 점검: "완료 버튼 = 트랜잭션 확정"과 persona_id 전량 태깅이 실제 구현/QA에서 보장되는지.
