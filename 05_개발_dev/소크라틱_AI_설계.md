# 소크라틱 AI 설계 (Socratic AI Pipeline)

- 작성일: 2026-07-01 · 작성: AAL 개발팀 · 핵심: "답을 주지 않고 사고를 끌어낸다"를 기술로 보장

## 1. 대화 파이프라인
```
학생 입력 → [상태 추정] 정체/오답/진전 판단
          → [발문 정책] 어떤 발문·비계 단계를 낼지 결정(과목 비종속 로직)
          → [페르소나 톤 주입] 선택된 길잡이 톤·표현으로 렌더
          → [가드레일 검사] 정답 유출·안전 위반 차단
          → 출력(스트리밍) + 로깅(persona_id·thinking_event)
```

## 2. 발문 정책 (Prompting Policy)
- 열린 질문 · 텍스트로 되돌리기 · 한 번에 하나. (UX 라이팅 가이드 §1과 일치)
- 정체 감지 시 **비계 사다리 단계 상향**: ①조건 표시 → ②결과 표시 → ③연결 힌트 → (그래도 막힘)난이도 하향·격려.
- 진전 시: Endorse(구체·근거 기반, 능력 아닌 노력·전략 칭찬).

## 3. 정답 유출 가드레일 (불가침) ★
- LLM 출력 후처리 검사: 정답 토큰/선택지 직접 노출·"정답은 X" 패턴 차단 → 재생성.
- 시스템 프롬프트: "학생이 스스로 도달하게 하라. 정답·완성 문장을 제공하지 마라." + 금지 예시.
- 오답에도 정답 대신 근거 문장으로 회귀 유도(수능형 문항 오답 시에도 동일).
- **검증팀 상시 QA 대상**(레드팀: 학생이 답을 유도해내려는 시도에도 버티는지).

## 4. 페르소나 톤 주입
- 공통 발문/비계 로직은 동일, **톤 파라미터**(어휘·리듬·Endorse 표현)만 페르소나별. 스킨이 아니라 실제 표현 차이.
- 4주 로테이션·A/B 위해 persona_id를 대화·세션에 전량 태깅.

## 5. 언어 운용 (EFL)
- 점화·심층 사고 발문은 한국어 허용, 근거·산출은 영어(커리큘럼 언어 원칙). 임계 어휘만 한국어 뜻풀이 지원.

## 6. 품질·안전
- 환각·부정확 억제: 지문 내 근거로 한정. 아동 안전(부적절 주제 차단), 자기비하·경쟁 유발 표현 필터.
- 응답 지연 최소화: 스트리밍 + 경량 상태추정.

## 7. 확장 훅
- 발문/비계 로직 과목 비종속 → 국어·수학 재사용. 톤·문구는 i18n·페르소나 파라미터로 지역·언어 변주.

> **결정 요청:** 가드레일을 "후처리 검사 + 프롬프트" 이중으로 갈지, 파일럿에서 레드팀 QA 주기.
