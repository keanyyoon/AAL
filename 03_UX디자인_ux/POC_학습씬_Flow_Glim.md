## 1. POC 콘텐츠 — 화성 탐사 (자체 제작 지문·문항)

### 지문 (English, 통제 자료 · 약 L3/B1)
**The Hard Landing on Mars**
> Mars is a hard place to land on. Its air is very thin, so it cannot slow a falling spacecraft much. Because of this, engineers must choose the landing method **depending on how heavy the rover is**. A **light** rover can be wrapped in airbags and bounce safely on the ground. But a **heavy** rover is too heavy for airbags, so it is lowered slowly by cables from a hovering "**sky crane**."

- 핵심 청크: `a hard place to land on` · `very thin air` · `slow a falling spacecraft` · `depending on how heavy` · `wrapped in airbags` · `lowered by cables` · `sky crane`
- 논리 구조(조건→결과): **가벼운 로버 → 에어백** / **무거운 로버 → 스카이 크레인** (문어 지문의 산호/모래→bumpy/smooth와 동일한 구조 → 같은 스캐폴딩 적용)

### 문항
- **Q1 (조건-결과 연결 · 의7/논6):** "무거운(heavy) 로버는 어떻게 착륙할까?" → 선택: `airbags` / `sky crane` (정답: sky crane)
- **Q2 (추론 · 논5):** "화성의 얇은 공기(thin air)가 왜 착륙을 어렵게 만들까?" (글에 직접 안 쓰인 함의 추론) → 예: "떨어지는 우주선을 충분히 감속시키지 못해서."

### 목표 스킬
의미(청크·`depending on` 구문) · 논리(조건-결과·추론) · 지식①(화성/우주 탐사 배경 활성화)

## 3. 화면 B — 학습 씬 (1-Pane + 하단 대화)
### 레이아웃 (태블릿 가로 · 세로로 쌓는 단일 pane)
```
┌───────────────────────────────────────────────────────────┐
│ ⟵            ✦ ✦ ✧ ○ ○   (별 2/5 켜짐)             Glim    │  ← 상단바(팝업 없음)
├───────────────────────────────────────────────────────────┤
│                                                           │
│   READING                                                 │
│   ╭ glass card (지문) ─────────────────────────────────╮   │  ← 콘텐츠 = 가장 밝음
│   │ The Hard Landing on Mars                            │   │
│   │ Mars is a hard place to land on. Its air is very    │   │
│   │ thin ... [depending on how heavy the rover is] ...  │   │  (점진 공개·하이라이트)
│   ╰─────────────────────────────────────────────────────╯   │
│                                                           │
│   QUESTION                                                │
│   무거운 로버는 어떻게 착륙할까?                              │
│     (  airbags  )   (  sky crane  )                       │  ← 선택지=빛(선택 시 가장자리 발광)
│                                                           │
│                 ··· 어둠/여백 ···                          │
│                                                           │
├───────────────────────────────────────────────────────────┤  ← 하단 대화 영역
│  (✷ sphere)  "무엇이 착륙 방법을 가르는 단서일까?"            │  ← AI 대사 : sphere를 대사 '좌측'에
│                                                           │
│                       "로버 무게요?"                        │  ← 학습자 대사 : '최하단'
└───────────────────────────────────────────────────────────┘
```
### 구성 원칙
- **1-Pane:** 지문 → 문항을 세로로 쌓음(2단 분할 없음). 위쪽이 콘텐츠, 아래쪽이 대화.
- **AI 대사(하단):** 좌측에 **반짝이는 추상 파형 sphere** + 오른쪽에 텍스트. 말할 때 sphere 파동이 살짝 살아나고, 끝나면 잔잔해짐.
- **학습자 대사(최하단):** AI 대사 아래 한 줄로, 방금 말한 내용이 텍스트로 뜸(우측 정렬).
- **마이크 버튼 없음:** 음성은 상시 감지(핸즈프리). 인식 중엔 sphere 근처에 미세한 청취 표시(점 파동)만, 버튼 UI 없음. (소음/실패 시에만 조용한 텍스트 입력 폴백 노출)
- 배경은 어둠·잔별·대각 광선으로 잔잔, 콘텐츠와 sphere만 빛난다.

## 4. 인터랙션 시퀀스 — 스캐폴딩 3기능 × Co-Thinking (Happy Path)
> 한 문항을 [접근]→[유발]→[대면]→도달로. 성공=별 하나 켜짐(스파클 1회). 대화는 하단, AI 좌측 sphere.

**Phase 0 · 진입 & 배경 활성화** [지식① / 예측하며 읽기]
- AI(sphere 잔잔, 하단): "지호야, 화성에 로봇을 내려놓는 게 왜 어려울까? 한번 상상해볼래?"
- 지문 제목+첫 문장만 밝아짐 → 예측 유도. 학습자 대사(최하단)에 자유 예측 표시. 채점 없음.

**Phase 1 · [접근] 도전 가능하게** (부담↓)
- 구조: 핵심 구문(`depending on how heavy`) 표시, 완성문제 페이딩(B-6).
- 문항 제시(1-pane 하단부 QUESTION): 응답형식 **고르기**부터(E-12). sphere는 개입 안 함(잔잔).

**Phase 2 · 학습자 1차 시도** (직접 판단)
- 학습자가 음성으로 답+이유(최하단에 텍스트로). 정답→Phase 3 자기설명 / 막힘→Phase 3 유관 지원.

**Phase 3 · [유발] 짐작·확인·파고들게** (자기설명 · 대화 수 사다리)
- 성공 시(T-3): sphere가 밝아지며 "왜 sky crane이라고 생각했어? 글의 어디가 그렇게 말하지?" → 근거 문장 복귀.
- 막힘 시(T-2 유관 + T-1 사다리): 하단 대사로 단계적:
  1) "무게 말고, 방법을 가르는 단서가 또 있을까?"
  2) "'depending on ___' 뒤에 뭐가 오는지 지문에서 찾아볼까?" (하이라이트 유도)
  3) "light면 airbags, heavy면 ___ ?" (빈칸)
  4) 부분 설명(정답은 학습자가 완성). 성공하면 지원 단계 낮춤(페이딩).

**Phase 4 · [대면] 자기 답 돌아보고 고치기** (오답 무해화)
- 오해 시 **빛으로 처리**: 맞는 연결(heavy↔sky crane)은 빛이 스며 자리 잡고, 틀린 연결(heavy↔airbags)은 빛이 사그라들며 되돌아옴. "틀렸다" 문구 없음.
- 변형: "다른 친구는 heavy도 airbags라고 했대 — 어디가 어긋났을까?"(결함답안 C-8, 부담↓).

**Phase 5 · 도달 & Co-Thinking 이어받기 + 페이딩**
- 스스로 "heavy → sky crane" 도달 → **별 하나 켜짐(스파클 1회).**
- 다음 문항 이어받기: Q2 추론 — "그럼 애초에 왜 착륙이 어려웠지? 얇은 공기랑 연결해볼래?" 다음엔 응답형식 고르기→쓰기로 상향(성장).

**Phase 6 · 마이크로 완료** (Small Win · Peak-End)
- 절제된 빛 번짐 1회 + "네가 스스로 빛을 켰어 ✦".
- 노력·전략 칭찬: "지난번 근거 1개 → 오늘 3개. 읽는 힘이 밝아졌어."
- 사고행동 로깅(텍스트 복귀·근거 대기·자기설명 질·비계 단계) = 학부모 리포트 원천.

## 5. 음성 인터랙션 (마이크 버튼 없음 · 상시)
```
학습자 발화(상시 감지) → STT → 최하단에 텍스트로 표시
   → 구조 레이어 판정(정답/근거/오해) → 대화 레이어 반응
      → 하단 AI 대사 갱신(좌측 sphere 파동 ↑) + TTS(짧게, 1~2문장) → 잔잔
```
- 마이크 버튼 없음: 핸즈프리. 청취 중엔 sphere 옆 미세 점파동만(버튼 UI 없음).
- AI 발화 짧게(수다 금지), 한 번에 하나의 질문, 학습자 말하는 동안 개입 안 함.
- 소음/인식 실패 시에만 조용한 텍스트 입력 폴백. 과잉 의인화 금지(AI=빛/파형 정체성).

## 6. Happy Path 한 줄
세션 선택(오늘의 빛·화성) → 진입·배경활성화 → [접근]문항(부담↓) → 1차 판단(음성, 최하단) → [유발]자기설명/유관 힌트로 스스로 도달 → [대면]오답은 빛의 사그라듦으로 교정 → heavy→sky crane 도달 = **별 하나 켜짐** → Q2가 판단 이어받음(페이딩) → 마이크로 완료(전략 칭찬·로깅).
