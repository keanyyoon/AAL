# UX 라이팅 가이드 (UX Writing) — 소크라틱 발문 · Endorse · 상태 문구

- 작성일: 2026-07-01 · 작성: AAL UX 디자인팀 (브랜딩 톤&보이스 협업)

## 1. 발문 (Socratic Prompts) — 답을 주지 않는다
- 원칙: 열린 질문, 텍스트로 되돌리기, 한 번에 하나.
- 예: "이 문장, 글의 어느 부분이 그렇게 말하는 것 같아?" / "여기서 '그것'은 뭘 가리킬까?" / "정말 그럴까? 근거가 어디 있지?"
- 금지: "정답은 B야" / "그건 틀렸어"(대신 "다시 글로 돌아가 볼까?").

## 2. 비계 사다리 문구 (점감)
1) "먼저 조건이 되는 부분에 표시해볼까?" → 2) "결과가 되는 부분도 찾아볼까?" → 3) "이 둘을 이으면 어떻게 될까?" (그래도 막히면 난이도 조정·격려)

## 3. Endorse (칭찬) — 구체·근거 기반
- 좋음: "지난번엔 근거가 1개였는데 오늘은 3개 찾았어. 그게 읽는 힘이야." 
- 피함: "천재다!" 같은 능력 칭찬(고정 마인드셋). → **노력·과정·전략** 칭찬.

## 4. 상태 문구
| 상태 | 문구(학생) |
|---|---|
| 빈(첫 진입) | "첫 탐험을 시작해볼까? 함께 밝혀보자." |
| 로딩 | "지도를 펼치는 중…" |
| 막힘 | "괜찮아, 서두르지 않아도 돼. 다시 글로 돌아가 보자." |
| 완료 | "오늘의 땅을 밝혔어! ✨ Lumen을 얻었어." |
| 에러 | "잠깐 길이 끊겼어. 이어서 다시 해보자(기록은 안전해)." |

## 5. 학부모 문구 — 담백·신뢰
- "오늘 지호는 B1 난이도 지문을 스스로 이해했어요. 추론형 문항에서 근거 제시가 늘었습니다." (과장·단정 회피, 관찰 기반)

## 6. 톤 규칙
- 반말/존댓말: 학생=친근한 반말(길잡이 동료), 학부모=정중한 존댓말.
- 이모지: 절제(성취 모먼트에만).
- 연령 적합·안전, 자기비하/경쟁 조장 표현 금지.

## 7. 확장 훅
- 모든 문구 i18n 키로 분리. 톤은 페르소나 파라미터로 변주(공통 로직 위).
