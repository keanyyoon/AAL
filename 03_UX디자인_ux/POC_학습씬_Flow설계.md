## 0. 설계 원칙 (POC에서 반드시 지킴)
1. **구조가 뼈대, 대화가 관절** — 과제 구조가 사고를 강제(가설 좁힘·이벤트·숙달 측정), 음성 대화는 그 안에서 "왜 골랐어?·자기설명·정서" 담당. [scaffolding p5]
2. **막히기 전엔 끼어들지 않는다** — Agent는 구석에서 대기, impasse(정말 막힘) 직전까지 지원 보류. [T-2 유관 스캐폴딩]
3. **정답 직행 금지** — 힌트→예시→부분해설의 단계적 지원(대화 수 사다리), 실력 늘면 지원 제거(페이딩). [ZPD/Scaffolding]
4. **스캐폴딩은 채점하지 않는다** — 점수 대신 데이터만 기록. 겁 없이 시도하게. [scaffolding p4]
5. **오답은 말이 아니라 화면 반응으로** — "오답입니다" 금지. 맞는 조각은 붙고 틀린 조각은 미끄러지듯. 무해화. [Dweck·Weiner]
6. **몰입 보호** — 세션 중 알림·팝업 금지, 화면 전환 최소, 에이전트 수다 금지. [Flow / Csikszentmihalyi]
7. **작은 성공·피크엔드** — 초반 난이도 낮게, 완료 순간 연출. 칭찬은 능력이 아닌 **노력·전략**. [Bandura·Dweck·Kahneman]
8. **인지부하 최소** — 한 화면 한 과업, 선택지 3~4개, 청킹. [Sweller·Hick·Miller]
9. **음성 우선 + 자막 병행** — 말+글 이중 채널, 음성 실패 시 텍스트 폴백. [Mayer / 접근성]

### 설계 포인트
- **추천 1 + 선택 3** = 결정 부담 최소(Hick). 추천은 ZPD 정중앙("딱 맞음")으로 강조(넛지). [Hick·Nudge]

- 진입 애니메이션 짧게 → 로딩은 "지도를 펼치는 중" 1초 이내.

## 2. 화면 B — 학습 씬 (Learning Scene) · 핵심

### 레이아웃 (태블릿 가로, 2-Zone)
```
┌───────────────────────────────────────────────────────────────┐
│ ◀ 나가기        진행 ●●○○○ (2/5)               🏛️ 도시기      │  ← 상단(팝업/알림 없음)
├──────────────────────────────────┬────────────────────────────┤
│  읽기 존 (좌 ~60%)                │  사고 존 (우 ~40%)          │
│                                  │                            │
│  The Hidden Octopus              │  ❓ 지금 할 일               │
│  To stay safe, the octopus       │  "산호 근처일 때 피부는?"    │
│  hides by [changing how it       │                            │
│  looks]. It can be bumpy or      │  ○ bumpy    ○ smooth        │  ← 응답 형식(고르기→쓰기 페이딩)
│  smooth, [depending on whether]  │                            │
│  it is near coral or near sand.  │  ┌───────────────────────┐  │
│                                  │  │ 🌙 길잡이 노바         │  │  ← Agent: 평소엔 조용히 대기
│  (점진 공개 A-1: 문장씩 열림)     │  │ (막힐 때만 다가옴)     │  │
│                                  │  └───────────────────────┘  │
│                                  │   [ 🎤 눌러서 말하기 ]      │  ← 음성 우선 + 자막
└──────────────────────────────────┴────────────────────────────┘
```
### 존 역할
- **읽기 존:** 저자극·집중. 하이라이트·점진 공개(A-1). 여기선 Agent가 말 걸지 않음.
- **사고 존:** 과제(구조 레이어) + Agent 대화(대화 레이어). Agent 아바타는 작게 구석에, impasse 전엔 정적.
- **음성 바:** 눌러서 말하기(PTT) → STT → 자막으로 되비침. 실패 시 텍스트 입력 폴백.

## 3. 학습 씬 인터랙션 시퀀스 — 스캐폴딩 3기능 × Co-Thinking (Happy Path)

> 한 문항을 **[접근] → [유발] → [대면] → 도달·페이딩**으로 운용. Co-thinking = 학생이 직접 판단(예측·선택·생성·설명)하고, 그 판단을 다음 단계가 이어받고, 익숙해지면 지원이 사라진다.

### Phase 0 · 진입 & 배경 활성화 [지식① / 예측하며 읽기 DRTA]
- Agent(음성, 짧게): "지호야, 지난번에 문어가 색을 바꿔 숨는 거 기억나? 오늘은 '어떻게' 숨는지 파고들 거야."
- 화면: 읽기 존에 제목+첫 문장만 노출(점진 공개 A-1). "무슨 얘기일 것 같아?" 예측 유도.
- 학생(음성): 자유 예측. → **채점 없음, 기록만.**

### Phase 1 · [접근] 도전을 가능하게 (부담만 덜고, 판단은 남김)
- 구조: 골격 우선 노출(A-2) 또는 완성문제 페이딩(B-6) — 문장을 다 보여주되 핵심 구문에 표시.
- 과제 제시(사고 존): "산호 근처일 때 피부는?" — 응답형식은 **고르기(최저 부담)**부터. [E-12 응답형식 페이딩]
- Agent: 개입하지 않고 대기. [막히기 전 안 끼어듦]

### Phase 2 · 학생 1차 시도 (직접 판단)
- 학생이 선택 또는 음성으로 답 + 이유 시도. 
- 정답이면 → Phase 3의 자기설명으로. 오답/무응답 지속(impasse 신호)이면 → Phase 3의 유관 지원.

### Phase 3 · [유발] 짐작·확인·파고들게 (대화 수 사다리 · 자기설명)
- **성공 시:** Agent가 답을 확정해주지 않고 **자기설명 유도**(T-3): "왜 bumpy라고 생각했어? 글의 어디가 그렇게 말하지?" → 학생이 근거 문장으로 복귀.
- **막힘 시:** 유관 스캐폴딩(T-2) + 대화 수 사다리(T-1) 단계적으로:
  1. "그밖엔 어떤 단서가 있을까?" (가장 약한 힌트)
  2. "조건이 되는 부분에 먼저 표시해볼까?" (텍스트로 회귀)
  3. "near coral / near sand 중 어디에 밑줄쳤어?" (빈칸형)
  4. (그래도 막히면) 부분 설명 — 단, 정답 문장은 끝까지 학생이 완성.
  → 성공하면 지원 단계를 **다시 낮춘다(fading).**

### Phase 4 · [대면] 자기 답을 돌아보고 고치기 (오답 무해화)
- 오해가 있었다면: 화면 반응으로 처리 — 맞는 조각(near coral↔bumpy)은 제자리에 붙고, 틀린 연결은 부드럽게 미끄러져 되돌아옴. **"틀렸다" 말 없음.**
- 생성 후 대조(B-4)/결함답안 평가(C-8) 변형: "다른 친구는 이렇게 풀었대. 어디가 어긋났을까?" — 내 답이 아니라 부담이 낮음.
- Agent 톤: 비난 0, 전략 귀인 — "접근을 살짝 바꿔보자." [Weiner]

### Phase 5 · 도달 & Co-Thinking 이어받기 + 페이딩
- 학생이 **스스로** "산호=bumpy, 모래=smooth"에 도달.
- Co-thinking: 이 판단을 **다음 문항이 이어받음** — "그럼 문어가 '움직이며' 숨는 것도 같은 원리일까?"(전이·다음 스텝).
- 다음 문항에선 응답형식을 고르기→쓰기로 올리고 힌트 시작 단계를 낮춤(지원 감소 = 성장 신호).

### Phase 6 · 마이크로 완료 (Small Win · Peak-End)
- 짧은 성취 연출(과함 금지): 빛 번짐 1회 + "근거를 스스로 찾았어 ✨".
- **노력·전략 칭찬:** "지난번엔 근거 1개, 오늘은 3개. 읽는 힘이 자란 거야." [Dweck]
- 사고행동 로깅(텍스트 복귀·근거 대기·자기설명 질·비계 단계). = 학부모 리포트 원천.

## 4. 음성 인터랙션 상세 (Agent ↔ 학습자)
```
학생 [🎤 PTT 눌러 말함]
  → STT 변환 → 자막으로 되비침(확인)
     → 구조 레이어가 판정(정답/근거 유무/오해 유형)
        → 대화 레이어가 반응 생성(발문·자기설명·유관 힌트)
           → Agent TTS 발화(짧게, 1~2문장) + 자막 동시
```
- **원칙:** Agent 발화는 짧게(수다 금지), 한 번에 하나의 질문. 학생이 말하는 동안 개입 안 함.
- **폴백:** 시끄럽거나 음성 인식 실패 → 텍스트 입력으로 자연 전환(강제 아님).
- **정서:** 머뭇거림·한숨 등 신호 감지 시 격려(관계성), 단 과잉 의인화 금지(AI 정체성 유지). [CASA·불쾌한 골짜기]

## 5. Happy Path 한 줄 요약
세션 선택(ZPD 추천) → 진입·배경 활성화 → [접근]문제 제시(부담↓) → 학생 1차 판단 → [유발]자기설명/유관 힌트로 스스로 도달 → [대면]오답 무해화·교정 → 도달 → 다음 문항이 판단 이어받음(페이딩) → 마이크로 완료(전략 칭찬·로깅).

## 6. POC 범위 밖(다음 단계)
- Unhappy path(연속 오답→난이도 하향, 음성 완전 실패, 이탈), 세션 종료 리포트 연결, 세션 전체(5문항) 오케스트레이션, 페르소나 5종 톤 분기.
