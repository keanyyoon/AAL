# Wanted Design System

React + CSS recreation of the **Wanted Design System** (원티드 디자인 시스템) — the open-source design system of [Wanted](https://www.wanted.co.kr), Korea's tech-career/recruiting platform. Source: the attached Figma community file "Wanted Design System (Community).fig" (distributed CC BY 4.0; file marker 2025-11-06). Credits from the file — Design: Hyungjin Kil, Chaeri Oh, Doeun Kim, Sanghyo Yee, Kyungmin Park, Sungho Cho, Jisoo Lee; Support: Shinhae Lee, Minjeong Kang, Seulgee Kim, Minju Park, Minsun Park, Miri Son, Hyunji Jeon.

**Products represented**: the Wanted job platform (web + iOS/Android app) and its sub-brands — Wanted Space, Wanted Gigs, Wanted Agent, Wanted LaaS, Wanted OneID, 원티드 채용 솔루션 (recruiting solution), 원티드 통합 로그인 (unified login).

## CONTENT FUNDAMENTALS
- **Bilingual**: Korean is primary, English secondary. Doc headings pair them ("Looking Forward / 기대하며").
- **Tone**: polite-formal Korean 해요체 — warm, invitational, never commanding. e.g. "새로운 개선 의견은 언제나 환영합니다", "댓글로 자유롭게 의견을 남겨주세요".
- **Person**: speaks as "우리(we)" about the team, addresses the reader indirectly; avoids "너/당신".
- **Casing (EN)**: Title Case for section names ("Table of Contents", "Before Use"); component names are plain Title Case ("List Cell", "Segmented Control").
- **No emoji** in product copy or docs. No exclamation-mark hype; statements are calm and factual.
- **Microcopy**: short noun phrases on buttons ("저장", "확인", "취소"); helper text is one full sentence with period.

## VISUAL FOUNDATIONS
- **Color**: dominant black/white surfaces. Primary blue `#0066FF` (rgb 0,102,255) is the only loud color; hover/pressed derive via Interaction overlays, light tint `#EAF2FE`. Text neutrals: `#171719` (strong), `#37383C` (normal, often at 61%–88% alpha), `#70737C` (neutral/assistive), `#AEB0B6`/`#C2C4C8` (disabled). Semantic: negative red `#FF4242`, positive green `#00BF40`, info cyan `#0098B2`, violet accents `#6541F2`/`#9747FF` (Figma annotations only). Full token set: Atomic (207) → Semantic "Ungrouped" (204) → Theme (55, light/dark) + Modal (13), Component (6), Frame (3) — all in `tokens/fig-tokens.css`, including **dark mode** scopes (`:root[data-theme="dark"]`) and responsive modes.
- **Type**: Pretendard JP for all UI; Wanted Sans (open-sourced by Wanted) for brand/display; SF Mono for code. 16-style scale (Title 1–3 / Heading 1–2 / Headline 1–2 / Body 1–2 Normal·Reading / Label 1–2 / Caption 1–2) in `tokens/typography.css` — exact letter-spacings like `-0.0253em`, `0.0057em`. Negative tracking above 17px, positive below.
- **Spacing/Layout**: 4px base; common paddings 8/12/16/20; viewport grid via Frame tokens (padding-horizontal 20 mobile). Cards and modals use Radius token (16–20); chips/buttons round at 8–10px (Round Buttons are full pills).
- **Corners**: squircle corner-smoothing in Figma; web uses plain border-radius. Checkbox 6px (small 4.5), cards 12–16, modal 20, thumbnails 8 (Radius=True).
- **Shadows**: rare and soft — `0 1px 4px rgba(0,0,0,.08)`, heavier `rgba(0,0,0,.12)` for popovers/menus; most elevation is done with hairline borders `inset 0 0 0 1px rgba(112,115,124,0.22)` instead of shadow.
- **Interaction states**: overlay components, not color swaps — Interaction/Normal|Light|Strong paint a black (or white on dark) veil at Hovered ~7.5%, Focused ~12%, Pressed ~13.5%. Disabled = 16–43% alpha content.
- **Backgrounds**: flat white/black; Gradient/Solid & Gradient/Multiple used only as protection masks over images (scroll fades, thumbnail overlays). No decorative gradients, no textures, no illustrations in the kit itself.
- **Motion**: minimal — Circular spinner (incl. Wanted-logo spinner), skeletons shimmer; no bounce.
- **Transparency/Blur**: dimmer `rgba(0,0,0,~52%)` behind modals; no glassmorphism.
- **Imagery**: photos only inside Thumbnail/Avatar with 1:1, 4:3, 16:9 Ratio helpers; neutral, product-real (company logos, people).

## ICONOGRAPHY
- Proprietary 24×24 stroke-rounded icon set (~126 glyphs), each with Fill/Thick/Tight/Small variant axes → **223 variants extracted to `assets/icons/icon-data.js`** (render with `assets/icons/Icon.jsx`, names in `Icon.d.ts`; paint with `currentColor`).
- Social/brand color glyphs (Google, Apple, Kakao, Naver…) and 5 bottom-nav glyphs (Recruit, Career, Social, My Page, Menu) included.
- No icon font, no emoji-as-icons. Icons default `#171719`/`#70737C`.
- Wanted logos (horizontal/vertical/square/symbol/favicon) are React components in `components/brand/`.

## INDEX
- `styles.css` — global entry (fonts → tokens → typography → per-group asset css).
- `tokens/` — `fonts.css` (Pretendard JP + Wanted Sans via jsDelivr CDN), `fig-tokens.css` (all 488 variables, light/dark/responsive modes), `typography.css` (16 text styles as vars + `.wds-*` classes).
- `assets/icons/` — icon-data.js + Icon.jsx (+ Icon.d.ts name index).
- `components/actions/` — Button (48-variant), Text/Icon/Outlined/FAB buttons, Chips (Chip, Action, Filter, Multi-Select), Action Area.
- `components/forms/` — Checkbox, Radio, Switch, Toggle Icon, Textfield, Select, Auto Complete, Segmented Control, Framed Style, Category.
- `components/content/` — Card, List Card, List Cell, Thumbnail, Avatar (+Group), Content Badge, Push/Value/Status badges, Section Header.
- `components/navigation/` — Tab, Pagination (Navigation/Dots), Bottom/Top Navigation, Menu, Footer, Essential.
- `components/feedback/` — Toast, Snackbar, Alert, Tooltip, Popover.
- `components/layout/` — Skeletons, Circular (+Wanted spinner), Divider, Status/Home/Navigation Bar, Safe Area, Ratio helpers.
- `components/decorate/` — Interaction overlays, Gradients, Dimmer, Opacity, Spacing, Modal resources, Scroll Bar, Filter Button.
- `components/legacy/` — v1 families still in the file (Round/Text/Outlined buttons, Person/Company/Academic avatars, Basic Divider, Custom Gradient, Overlay, Bookmark).
- `components/brand/` — Wanted logo set.
- `guidelines/` + `ui_kits/wanted-web/` — specimen cards & sample screen (see Design System tab).
- `SKILL.md` — agent skill entry point.

## Notes & intentional gaps
- **No logo bitmap asset was shipped separately** — logos exist as extracted vector components in `components/brand/`.
- Fonts load from jsDelivr CDN (`Pretendard JP`, `Wanted Sans`); if offline use is needed, ask the user for font binaries.
- Sub-brand logotypes (Space/Gigs/Agent horizontal marks) did not resolve to unique nodes during extraction; re-extract on request.
- The file's icon glyph bodies for a few wrappers reference an external library node (renders empty slot); all main glyphs extracted fine.
- Component naming follows Figma family names flattened to PascalCase (e.g. `Button/Button` → `ButtonButton`). Runtime namespace: `window.WantedDesignSystem_315979`.

## Coverage accounting
The Figma file lists 959 component "families", but that number counts every variant-axis set separately across two coexisting generations of the system plus internal parts. What this project builds is the complete set of ~130 unique public families of the current generation, plus the v1 families that differ visually (in `components/legacy/`), plus the full ~126-glyph icon set (as `assets/icons/icon-data.js`, 223 entries — icons are data, not per-glyph .jsx, by design). Intentionally not rebuilt as separate components: (a) duplicate definitions of the same family on legacy pages (Theme/Element/Basic/Component2 repeat Button/Text, Divider, Checkbox, etc.), (b) internal `*/Resource/*` sub-parts that only exist to compose their parents (many ARE included where parents needed them), (c) Figma-documentation aids (`Arrow with Texts`, `Inspect/Measure`, `_Dummy`, `_Ratio`, `.Color/Overlay` annotation helpers, `_Theme/Primary Button` demo), and (d) per-glyph icon variant sets, covered by icon-data. Ask if you want any specific family from those groups materialized as its own component.

Skipped-family enumeration (why each category is intentional): **icons** — every "Icon/Normal/*", "Icon/Color/*", "Icon/Navigation/*" glyph set and their page-duplicates ("Bookmark", "Bubble", "Camera", "Bell", "Chevron *", "Circle *", "Arrow *", …) ship as `assets/icons/icon-data.js` entries instead of per-glyph .jsx; **legacy duplicates** — same-named sets re-defined on old pages ("Button/Text", "Basic/Divider", "Control/Checkbox (3-variant)", "Chip/Action (12-variant)", "Badge/Push", "Content Badge", "Avatar/Person (10-variant)", "Divider", "Bookmark (활성화)", "badge (상태)"); **internal resources** — "*/Resource/*" sub-parts ("Auto Complete/Resource/Item/Cell", "Bottom Navigation/Resource/Tab/Android|iOS|Web", "Cell/Resource/Trailing Content/*", "Card/Resource/*", "Modal/Resource/*", …) exist only to compose parents and are already baked into the built parents (several that parents import are compiled individually); **doc annotation aids** — "_Dummy", "_Ratio", "_Mask - Squircle", "Arrow with Texts(- Algorithm)", "Inspect/Measure", ".Color/Overlay", "agent alt 3", "Content (상태)", "Custom Gradient", "Blank (Color)" and the /Work example content; **deprecated** — "Status Bar/_Status Bar (Deprecated)".
